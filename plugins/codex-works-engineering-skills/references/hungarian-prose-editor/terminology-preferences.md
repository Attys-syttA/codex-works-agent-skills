# Terminology Preferences

These are general editorial preferences, not institution-specific terminology.
Always preserve mandatory terms from the user's source.

## Prefer concrete verbs

| Avoid when possible | Prefer |
|---|---|
| megvalósításra kerül | megvalósul / elvégzik |
| benyújtásra kerül | benyújtják |
| ellenőrzés lefolytatása | ellenőrzés / ellenőrzik |
| tájékoztatás adása | tájékoztatják |
| intézkedés megtétele | intézkednek |

## Useful official wording

| Heavy wording | Clearer wording |
|---|---|
| tekintettel arra, hogy | mivel |
| annak érdekében, hogy | azért, hogy |
| abban az esetben, ha | ha |
| a továbbiakban | ezután / később |
| haladéktalanul | azonnal / késedelem nélkül, if required |

## Technical wording

- Keep `API`, `JSON`, `YAML`, `hash`, `commit`, `endpoint`, `runner`, `workflow`, and similar established terms when they are identifiers or common technical terms.
- Translate only surrounding prose.
- Prefer "hiba", "kockázat", "hatás", "bizonyíték", "javítás", "ellenőrzés" for report structure.
- Use "feltételezés" for hypothesis and "megfigyelés" for observed evidence.

## Risk and threat wording

| Vague or awkward | Prefer when supported |
|---|---|
| számítógép-ellenőrzést fenyeget | a számítógép zárolásával fenyeget |
| megkényszerítés | kényszerítés / zsarolás |
| rosszindulatú szándékú tartalom | támadó vagy megtévesztő tartalom |
| pénzügyi manipulációs jelleg | fizetési csalásra utaló kérés |
